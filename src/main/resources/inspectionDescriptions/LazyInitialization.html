Detects method calls that trigger lazy initialization on entity getters outside of an active session, which can lead to LazyInitializationException.

This inspection analyzes method calls to detect cases where a lazy-loaded entity is accessed without an active persistence context. If a method fetches an entity field using a getter and the entity is not initialized within a transactional or Open Session in View (OSIV) context, the inspection highlights the call as a potential source of runtime errors.

Example: 


    @Entity
    class User {
        @OneToMany(fetch = FetchType.LAZY)
        private List<Order> orders;
    }

    void processUser(User user) {
        List<Order> orders = user.getOrders(); // May cause LazyInitializationException
    }

You can enable and disable OSIV mode in plugin settings tool window.
