Detects methods annotated with @Transactional(propagation = Propagation.MANDATORY) that are called without an existing transaction, which can lead to runtime errors.

This inspection checks whether a method marked with Propagation.MANDATORY is invoked within an active transaction. If no transaction is present at the call site, the inspection highlights the annotation and suggests adding a transaction at a higher level.

Example: 


    @Transactional(propagation = Propagation.MANDATORY)
    void saveData() {
        // Some database operation
    }

    void process() {
        saveData(); // This will cause an error if no transaction is active
    }
